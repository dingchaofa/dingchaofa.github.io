1.好用的nodemon工具自启动

Monitor for any changes in your node.js application and automatically restart the server - perfect for development
https://github.com/remy/nodemon
检测文件的改变并且自启动，注意启动文件时，需要通过node启动入口文件，不然的话，程序就会默认执行package.json中的start命令。

2. es6的import

这种引入模块的方法有很多种方式

  1. import { Nuxt, Builder } from 'nuxt' 需要多少方法，就引入多少
  2. import * as circle from './circle';，这种是所有的方法挂载在circle上，适合在自己写的模块，所有的方法自己都知道。

注意这两者引用之间的区别。

这里说一下，node.js中的exports，有时候暴露接口会用混淆。
Node。js遵守commonJs规范，暴露接口是通过

  1. module.exports //是默认导出
  2. exports.value //是导出某个方法，如果这两者同时存在，只导出module.exports的结果
  3. exports = value // 无法导出模块
  4. 如果想要暴露的模块接口，没有用module.exports或者exports.value模式，则导出空对象

http://es6.ruanyifeng.com/#docs/module
http://www.ghostchina.com/module-exports-and-exports-in-node-js/

3. sha1（Secure hash Aigo）安全散列算法

微信服务器对请求的数据进行sha1算法加密，开发者同样验证数据是否来源于微信，而决定如何处理。

开发者通过检验signature对请求进行校验（下面有校验方式）。若确认此次GET请求来自微信服务器，请原样返回echostr参数内容，则接入生效，成为开发者成功，否则接入失败。加密/校验流程如下：

1）将token、timestamp、nonce三个参数进行字典序排序 2）将三个参数字符串拼接成一个字符串进行sha1加密 3）开发者获得加密后的字符串可与signature对比，标识该请求来源于微信

4. raw-body

https://www.npmjs.com/package/raw-body
是一个把http请求体，解析成需要的数据格式的库。

var getRawBody = require(‘raw-body’)
getRawBody(stream, [options], [callback])，如果全局promise存在并且没有指定回调函数，则返回一个promise。

  1. stream：是请求体的内容

  2. options：

    • length-与请求体的长度相匹配，否则返回400error，表示由于语法无效，服务器无法理解该请求，客户端不应该在未经修改的情况下重复此请求。
    • limit-设置请求体的返回大小界限，如果返回的大小大于这个界限值就返回一个413error code，表示请求主体的大小超过了服务器规定的限度，服务器可以选择关闭连接或者返回 Retry-After 首部字段。
    • encoding-编码，解析请求体的编码方式。如果没有指定则默认是buffer类型数据。

    5. xml2js

    https://www.npmjs.com/package/xml2js
    这个库可以把xml格式解析成js对象，也可以把js对象解析成xml格式文件。

注：有时候传入的数据类型的可能不是期望的，例如buffer类型数据，Node.js会自动解析成能够解析的数据格式。

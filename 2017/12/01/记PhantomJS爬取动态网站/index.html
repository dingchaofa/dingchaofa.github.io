记PhantomJS爬取动态网站

Dec 01, 2017js库点击

1. 前言

有关PantomJS的介绍，详细请查阅官网：http://phantomjs.org/，本篇记述自己在使用PhantomJS爬去LOL官网人物数据所遇到的一些坑。

爬取静态网站数据，直接可以使用cheerio，可以参见我的另一篇关于静态网站的爬虫。动态网站由于涉及到ajax、js填充页面等，不是后台模版的数据，因此只能模拟浏览器环境，加载js，再提取我们所需要的数据，这里就要用到PantomJS。

自己在本次爬去LOL官网人物数据的核心过程是，通过PhantomJS模拟浏览器环境，找到数据接口，在evaluate回调函数内，也就是window环境，执行数据接口，获取数据，通过fs模块写入文件。

2. 安装

虽然安装，看别人的教程 `npm i phantomjs -g`即可，但是放在自己系统上就是各种error，error到头疼。废话少说，不折腾，macOS系统直接用`brew install phantomjs`，全局安装。npm各种报错。总之是全局安装，为此，之后我也改用yarn来管理包了。

2. 分析

爬取需要的数据，根据自己的踩的坑，总结一下可能会踩的坑。

> 1. 首先个人觉得PantomJS不太好用，因为调试什么的，有时候会没有反应，当然也存在自己水平低的问题。针对自己的问题，首先不考虑使用PhantomJS，数据既然是js添加的，那么就会存在接口。一般开发者都会封装好接口调用，通过浏览器的控制台的network，找到相关资源的initiator，js也没几行，应该能分析得到。找到接口，如果能获取需要的数据，就不要再用PhantomJS了。
然而我想要想要获取人物更加详细的信息，就需要针对不同的URI，来抓取。

2. js添加的数据，即使通过setTimeout在evaluate函数内通过DOM操作获取，有时候也加载不出来。那就试试第二种方法，接口，反正所有页面加载数据接口都是一样的。

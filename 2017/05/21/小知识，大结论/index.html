1、关于函数中常量的存在

今天看闭包时想到这么一个问题，函数中的常量怎么在函数外调用。问题看似简单，但是我们延伸一下

注：测试是在不同的开发工具页面下进行的，因为当你执行一次测试时，浏览器会记住这个测试的结果，对你下次测试结果有影响。

2.let,var对window对象的name属性的赋值

今天碰到一个尴尬的问题

首先window对象本身自带name属性，DOM对象没有name属性，有localName属性。

研究了一番，let由于增加了块作用域的概念，所以let name=’ccc’，不会更改window.name的值，this指向window。

var 没有块作用域的概念，在顶层对象中，var声明的对象在所有子作用域中都可以调用。

由于this指向全局作用域，name的值在fn2中被更改，所以是’aaa’。

3.声明提升和new的问题

说了多少次的声明提升，总是记不住。

这道题记住两个原则和new一个对象的过程

  1. 函数和var 声明的都会提升到作用域的最顶部。

  2. new一个对象时，this指向实例化的对象。（实例化就是new的结果）

    例如：当代码 new foo(...) 执行时：

    1. 一个新对象被创建。它继承自foo.prototype. //foo的原型
    2. 构造函数 foo 被执行。执行的时候，相应的传参会被传入，同时上下文(this)会被指定为这个新实例。new foo 等同于 new foo(), 只能用在不传递任何参数的情况。
    3. 如果构造函数返回了一个“对象”，那么这个对象会取代整个new出来的结果。如果构造函数没有返回对象，那么new出来的结果为步骤1创建的对象，ps：一般情况下构造函数不返回任何值，不过用户如果想覆盖这个返回值，可以自己选择返回一个普通对象来覆盖。当然，返回数组也会覆盖，因为数组也是对象。

结合这道题看是怎么new的；

结果1，没什么好说的，函数也是对象，这个函数的对象有一个getName方法。

结果2，就是声明提升了，var getName和function getName(){…}被提升到作用域的顶部。但是呢，由于var getName在第4句被赋值了，故而结果就是第四句的函数执行的结果。

结果3，先执行Foo()，由于之前声明过getName，所以现在getName = function (a) { alert (1); };这个this指向window，而window中的getName就是 function (a) { alert (1); }。

结果4，在window作用域中运行，当然还是 function (a) { alert (1); }了。

结果5，注意new的对象，对象是Foo.getName()，也就是Foo对象的getName方法。

结果6，这个new的对象是Foo().getName()，由于这里会执行Foo函数，会有this。前面提到this指向自身，this.getName()其实就是找Foo的原型上getName方法。new的是Foo.prototype.getName = function (a,b,c) { alert (3);};

结果7，你再new一次，this还是指向原型。

最后：我为什么每个getName都有参数呢？你可以console.dir(Foo)看看，会让你知道哪个是哪个getName。是原型上的，还是只有属性。函数是有length的。

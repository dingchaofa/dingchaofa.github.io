1. 微信浏览器下，正常API无法刷新

真是一大坑，微信浏览器下，使用：

window.location.reload()或window.location.href = window.location.href是无法页面内刷新的，搜到的办法是修改查询参数，可能是微信因为自带刷新按钮故而对这两个API的支持不友好。

2. 利用svg制作运动轨迹

有些页面需要UI做特殊轨迹运动，弯弯绕的轨迹用即使是用jquery写也是很麻烦，而且也达不到光滑运动的效果。下面这个函数既可以让元素按照轨迹运动，还可以随着运动的曲率改变元素的方向，网上关于这个教程很多，我也是摘抄的。

3. 有意思的transition

有一个需求是写一束束光线（都是png），沿着一个方向从一端在屏幕上不断的运动到另一端，就像下雨一样，需求很好理解。但是如何用最简单的方式实现呢？嗯，从a点到b点中间加个过渡效果就好了。原理是如此的简单，但是他的精髓，这才是我想要说的。看代码：

浏览器的异步机制使得我们可以利用transition产生位置突变时的过渡效果。
试着互换ele.style.transform = 'translateY(1138px)'和ele.style.transform = 'translateY(' + initY + 'px)'的位置，还能改变运动的方向，单向运动，双向运动，亲眼看到才真的会觉得有意思。

4. canvas貌似会让浏览器硬件加速。

遇到一个软需求，就是页面播放完视频，退出之后会有短暂的黑屏（具体时间视手机性能而定）。由于android是全屏播放的，当然有些大公司在腾讯的白名单上（这个白名单的作用就是让视频在android系统的微信下 播放形式和iOS是一样的，即不全屏播放。这个功能是收费的，说来也是垄断恶，谁让人家微信用户那么多呢。）如果你不在也没关系，但是会有一个小尴尬，就是那个瞬间闪黑。突然有一个项目竟然没有闪黑，我简直是惊呆了，因为在网上查了很多都没有解决的办法，之前还给客户说目前的技术没法解决（现在想想内心是多尴尬）。
废话少说，解决的办法就是在页面里加一个canvas标签，无论你用不用，即使你display：none，他也会发挥作用。啊，“内牛满面”。不知有多少像我这样的基础程序员忍受着这个问题的毒害。这篇也做不了SEO，如果能让大家知道就好了。
至于为什么能解决，我搜了关于canvas对浏览器的性能影响，也没有硬件加速这种说法。自己实力不济，很多原理都没法明白，顶多会用，距离成为一个前端大牛，还有很长的路要走。

移动端项目感想

说来移动端相较于pc端还要容易些，他们的区别也没啥能遇到的问题一般都能解决，在这个过程中成长。其实自己写这么多项目基本上是不用jquery，除了一些需要用到ajax的地方。感觉在技术水平上也没啥太大的成长，移动端也不是一个新领域。自己对js的使用和浏览器的性能，写的时候也有很多不懂，一直想着编写高性能的代码。没事还是多看看把《JS高程》吧。实力真是太弱了。

Linux服务器配置（二）

Aug 17, 2017命令行点击

开启防火墙

  1. 清除默认防火墙设置
    sudo iptables -F

  2. sudo vi /etc/iptables.up.rules新建防火墙

  1. sudo iptables-restore < /etc/iptables.up.rules指定防火墙的位置

  2. sudo ufw status查看防火墙状态
    显示Status: inactive，则是未激活

  3. sudo ufw enable激活防火墙
    Command may disrupt existing ssh connections. Proceed with operation (y|n)? y
    防火墙即可启动

  4. 开机自动启动防火墙
    sudo vi /etc/network/if-up.d/iptables

  1. 提高其执行权限
    sudo chmod +x /etc/network/if-up.d/iptables

安装fail2ban

fail2ban 是 Linux 上的一个著名的入侵保护的开源框架，它会监控多个系统的日志文件，并根据检测到的任何可疑的行为自动触发不同的防御动作。事实上，fail2ban 在防御对SSH服务器的暴力密码破解上非常有用。
参考：https://linux.cn/article-5067-1.html

  1. sudo apt-get install fail2ban

  2. sudo vi /etc/fail2ban/jail.conf配置一些专有的设置

  3. sudo service fail2ban status查看fail2ban运行状态，可通过 sudo service fail2ban stop和sudo service fail2ban start停止和开启fail2ban服务

安装nginx

  1. sudo apt-get install nginx
  2. cd /etc/nginx , cd conf.d,sudo vi dingcf-8081.conf

  4. sudo nginx -t检查配置语法是否正确

  5. sudo nginx -s reload重启nginx
    之后就可以打开这个120.79.89.232地址，直接访问8081端口。
    通过取消注释nginx.conf中的server_tokens off;，就可以隐藏响应头里的nginx版本。

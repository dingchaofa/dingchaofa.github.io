记移动端项目笔记1

1. 简介

这个项目没有用到什么牛逼的技术，只是一些DOM操作，但是其中有些函数的实现颇引发了我的一些思考，以及一些效果的实现，故而记录下来，以致自己后续的思考。这也不是我的第一个项目，算是对过去几个项目的学习总结。

2. <meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">

没有遇到，如果只是读文档，理解的还是不够，现在的手机屏幕种类繁多，做适配是一个必须的需求，之前我还用动态REM方案来实现，直到遇到<meta name="viewport" content="width=640, user-scalable=no, target-densitydpi=device-dpi">，啊啊啊，原来这么方便呀，640是iPhone6s plus的物理像素，也是设计师设计稿经常采用的像素宽度，这样一来，我要求设计师给640*1138的设计稿，css直接就可以用px（像素）写了，设计稿是多少，就用多少像素。可以做到设计稿的像素级实现。也不用担心手机的适配问题了。

3. 可能是浏览器的原因，应该是浏览器的原因

加上上面的那个标签后，手机端上会有1px的高度不足，在浏览器底部表现为屏幕有1px的白色。
这个不仅是手机上有，在Chrome 手机模式开发者工具下也是同样的。具体原因没有找到，解决方式是给body加相应的背景色。

4. 判断PC和移动端

通过判断是pc还是移动端，针对不同的需求做区别。如果是pc端，会在<html>加上<html class="pc">，移动端自然就是<html class="mobile">。

5. 移动端横屏观看

有写页面是为了横屏浏览，但是用户的习惯大多都是竖着看，android用户一般都会锁屏，所以即使是横屏也能正常观看，某些android手机，在微信里即使不锁屏，页面也不会随屏幕横过来就切换屏幕方向。这个时候就要给用户竖屏锁屏浏览提示了。

window.addEventListener("orientationchange", orientationchange, false)

function orientationchange() {
    if(window.orientation == 180 || window.orientation == 0) {
        //竖屏代码处理
    }
    if(window.orientation == 90 || window.orientation == -90) {
        //横屏
        if(browser.versions.iPhone || browser.versions.iPad || browser.versions.ios) {
            //iOS
        } else {
            //android
        }
    }
}

6. 浏览器私有前缀

在js文件里写css3的时候，要加一些浏览器的私有前缀

function perfixerCss(element, key, value) {
    ["Moz", "O", "Ms", "Webkit", ""].forEach(function (prefix) {
        element.style[prefix + key] = value;
    }     );
}

7. touchend事件没有e.targetTouchs方法

常见的移动端三个事件触摸事件，只有touchend事件没有e.targetTouchs方法，可以选择e.changeTouchs方法

8. 判断iOS和android

移动端做iOS的兼容真的很头疼，虽然都是webkit内核，但是iOS就是不一样，视频的播放形式不一样还好说，可是最令人意想不到的是它对绝对定位的支持，网上有很多关于绝对定位时，调出输入法挡住输入框的方法，这里也记一个element.scrollIntoView(false);，这个要在能滚动的元素上加，否则无效。
